## OrcaNet — In Progress

**Done:**
- Package scaffold: full module skeleton, `pyproject.toml`, multi-stage Dockerfile, Typer CLI (`serve` and `version` commands), Hydra config (`config.yaml`, `retriever/hybrid.yaml`, `embedder/cross_domain.yaml`, `llm/openai.yaml`), workspace registration in root `pyproject.toml`, full Docker Compose service wiring (depends on postgres, orcamind, orcalab with healthchecks, OPENAI_API_KEY passthrough), notebook scaffold with `.gitignore` override to preserve the cross-domain transfer demo notebook, unit test suite (package importability, parametrized submodule tests, per-test CLI runner fixture, pyproject.toml anchor config path resolution)
- Domain-adversarial cross-domain embedder: `CrossDomainEmbedder` (Ganin et al. 2016) maps 25-dim statistical meta-features to a 64-dim domain-invariant embedding space; shared `_FeatureMLP` [128→64] trunk feeds a task-type classifier and an adversarial domain classifier separated by `GradientReversalLayer` with mutable `alpha`; `fit()` optimises `L_task + λ·L_domain` under the Ganin progressive schedule (`α(p) = 2/(1+exp(−10p))−1`); `embed()` returns L2-normalised unit-sphere vectors for cosine similarity retrieval and safely restores the caller's training mode in a `finally` block; `save()`/`load()` checkpoint config + `state_dict()` with `weights_only=True`; public API exposed from `orcanet.embeddings` via a lazy `__getattr__` shim that defers `import torch` until first access; 15-test unit suite covering GRL gradient negation, alpha scaling, output shape, L2 normalisation, training-mode preservation, DANN convergence, domain invariance dispersion ratio, and persistence roundtrip
- Text-based task description embedder: `TextTaskEmbedder` encodes natural language task descriptions into 384-dim L2-normalised semantic vectors via `sentence-transformers/all-MiniLM-L6-v2`; three fusion strategies (`concat`, `add`, `attention`) combine the text vector with 25-dim `StatisticalEmbedder` meta-features into a configurable `output_dim`-dim fused embedding; `embed_from_description()` returns a standalone 384-dim text vector; `embed_with_stats()` fuses text and statistical features under a `torch.no_grad()` context; `embed_batch_descriptions()` batch-encodes a list of descriptions in a single forward pass; all outputs L2-normalised for cosine similarity retrieval; public API exposed from `orcanet.embeddings` via the lazy `__getattr__` shim; 16-test unit suite (4 classes) covering shape and normalisation correctness, all fusion strategies, semantic similarity ordering across domains, identical-description identity, and batch/individual numerical equivalence; offline `_DeterministicSentenceTransformer` stub with a 36-keyword domain vocabulary enables the full test suite without a network connection or a local model cache
- GNN-based neural architecture embedder: `ArchitectureEmbedder` (an `nn.Module`) encodes any model architecture config dict into a 32-dimensional L2-normalised embedding via residual adjacency-matrix message passing; `ArchitectureGraph.from_model_config()` converts the config into GNN-ready tensors — 16-dim node features per layer (8 one-hot layer type + log-scaled size normalised to `[0,1]` + 7 one-hot activation), a `(2, n_edges)` int64 `edge_index` for sequential and validated skip-connection edges, and a `(3,)` global feature array; three residual message-passing rounds (`h ← h + W(Âh)`, where `Â` is row-normalised with self-loops) prevent averaging collapse on sequential graphs; `similarity()` computes cosine similarity as a dot product of unit vectors; `find_similar_architectures()` ranks a candidate pool by similarity with configurable `top_k`, raising `ValueError` on negative inputs; optional `torch-geometric` `GCNConv` path available via `pip install orcanet[gnn]` without any API change; all tensor allocations track the module device so CUDA deployment works without modification; input sanitisation guards negative layer sizes (clamped to 0) and malformed skip-connection entries (skipped with debug log); `ArchitectureEmbedder` and `ArchitectureGraph` added to `orcanet.embeddings.__all__` and the lazy `__getattr__` shim; 45-test unit suite across 6 classes covering graph construction (node counts, feature encoding for all layer types and activations, sequential and skip-connection edges, global statistics, degenerate empty config), embedder correctness (shape, L2 norm, dtype, training-mode preservation in both directions, determinism across calls), similarity (identical configs → 1.0, self-similarity exceeds cross-architecture similarity under fixed seed, return type, symmetry), and retrieval (top-k count and sort order, fewer-candidates-than-k, tuple types, identical query is top result, boundary cases `top_k=0` and `top_k=-1`)

- CKA-based feature transfer strategy: `linear_cka(X, Y)` pure-numpy implementation of Kornblith et al. 2019 (`CKA = ||Y_c^T X_c||_F^2 / (||X_c^T X_c||_F · ||Y_c^T Y_c||_F)`); `FeatureTransfer` concrete `TransferStrategy` — forward-hook activation collection on shared `probe_data` with `try/finally` cleanup and per-model device handling; depth-ordered layer sorting; depth-weighted `1/(i+1)` overall score; configurable `cka_threshold` selects `recommended_layers`; `execute_transfer` clones the registered target model and patches matching-shape parameters for every recommended layer; `TransferStrategy` ABC and `TransferScore` dataclass shared across all strategies; 37 unit tests covering CKA correctness, FeatureTransfer scoring (identical/random models), guards, metadata, structural invariants, and execute_transfer
- Direct weight transfer strategy: `WeightTransfer` transfers parameter tensors by matching on `match_by` = `"name"`, `"shape"`, or `"both"`; `_find_source_tensor` resolves the source tensor with a shape-safety check so `copy_()` is never called unsafely; `execute_transfer` starts from a `deepcopy` of the registered target model, copies matched source weights in, and reinitialises unmatched parameters via `_safe_reinit` (`kaiming_uniform_` for 2-D+ tensors, `zeros_` for 1-D biases); transferred parameter names are stored in `last_transferred` for direct use with `get_optimizer_with_layer_lr`; `get_optimizer_with_layer_lr` builds a per-parameter Adam optimizer with `base_lr * decay` for transferred layers and `base_lr` for new/reinitialised layers; `score_transfer` returns binary per-parameter `layer_scores` (1.0/0.0) and `overall = n_matched / n_total`; 36 unit tests covering scoring across all three match_by modes, execute_transfer target-registry semantics, shape-mismatch reinit path, optimizer LR group correctness, guards, and metadata; `WeightTransfer` and `get_optimizer_with_layer_lr` added to `orcanet.transfer` public exports
- Architecture transfer strategy: `ArchitectureTransfer` recommends and adapts architectures for a target domain by comparing architecture graph embeddings; `adapt_architecture` deep-copies a config and updates `input_dim` and last-layer size from the target task's `n_features`/`n_classes`; `_build_sequential_from_config` materialises an `ArchConfig` (with `relu`, `sigmoid`, `tanh`, `gelu` activation support) into an `nn.Sequential`; `score_transfer` calls `OrcaMindClient.get_best_model` asynchronously (via `_run_coro` sync/async bridge) and ranks all registered candidate configs by cosine similarity; `execute_transfer` initialises boundary layers with `kaiming_uniform_`/`zeros_` and copies middle-layer weights from the source model where shapes match; `recommended_layers` is always `[]` (architecture-level decision); 29 unit tests covering `adapt_architecture` (including None-guard and original-immutability cases), score_transfer (OrcaMind mock call, embedder usage, overall/layer_scores invariants, empty-registry edge case), execute_transfer (input/output/middle dimensions, source immutability, lazy score call, all four activation types), metadata, and config registry guards; `ArchitectureTransfer` and `adapt_architecture` added to `orcanet.transfer` public exports
- Multi-task transfer strategy: `MultiTaskModel(nn.Module)` — shared backbone with per-task output heads stored in `nn.ModuleDict` so heads appear in `parameters()` and `state_dict()`; two loss paths: `compute_loss(batch, weights)` for equal/gradnorm weighting and `compute_uncertainty_loss(batch)` implementing the Kendall et al. 2018 learnable log-variance objective (`L = Σ exp(−2·log_σᵢ)·CEᵢ + log_σᵢ`); `log_sigmas` stored in `nn.ParameterDict` so uncertainty parameters are optimised by a standard Adam step; `MultiTaskTransfer(TransferStrategy)` — `add_task(task, head_output_dim)` creates a two-layer head (`Linear → ReLU → Linear`) and re-balances `_task_weights` after every registration; three weighting schemes: `"equal"` (uniform 1/n), `"uncertainty"` (per-task learnable log-variance, initialised to zero for equal initial weighting), `"gradnorm"` (uniform init, caller updates via `update_gradnorm_weights(grad_norms)`); `score_transfer` computes cosine similarity between `CrossDomainEmbedder`-embedded task feature tensors registered via `register_task_features`, returning a neutral 0.5 score when no features are provided; `execute_transfer` auto-registers tasks from `task.n_classes` when not already registered and returns a `MultiTaskModel` bundling the shared backbone with all registered heads and log-sigma parameters; backbone output dimensionality inferred from the last `nn.Linear` in the backbone via `_get_backbone_out_dim`; `MultiTaskModel` and `MultiTaskTransfer` added to `orcanet.transfer` public exports; 56 unit tests across `TestGetBackboneOutDim` (4), `TestMultiTaskModelForward` (6), `TestMultiTaskModelLoss` (5), `TestMultiTaskModelUncertaintyLoss` (4), `TestAddTask` (9), `TestScoreTransfer` (8), `TestExecuteTransfer` (7), `TestTransferMetadata` (7), `TestGradnormWeighting` (4), `TestUncertaintyWeighting` (2) — **158 tests total** across the transfer module

- Hybrid task retrieval pipeline: three-stage async retrieval system that narrows from broad vector similarity to a small set of semantically ranked candidates. `QueryExpander` (`orcanet.retrieval.query_expander`) takes any `langchain_core.language_models.BaseLLM` and produces `n_expansions` (default 3) alternative task descriptions via `ainvoke`; `_parse_list_from_response` strips numbered prefixes, bullet markers (`-`, `*`, `•`), and blank lines from the LLM response, returning only non-empty strings. `LLMRanker` (`orcanet.retrieval.ranker`) re-ranks a `list[Task]` against a query `Task` — it builds a structured prompt from both query and candidate metadata, calls `ainvoke`, strips optional markdown backtick fences, and validates the JSON body against `_RankedList(rankings: list[_RankedItem])` where `_RankedItem` enforces `score: float` in `[0.0, 1.0]` via `pydantic.Field(ge=0.0, le=1.0)`; any `ValidationError`, `json.JSONDecodeError`, or `ValueError` returns `[]` with a `WARNING` log rather than raising; results are sorted by score descending and truncated to `top_k`. `HybridRetriever` (`orcanet.retrieval.retriever`) accepts `faiss_index`, `task_repository: TaskRepository`, `embedder: CrossDomainEmbedder`, `query_expander: QueryExpander`, `llm_ranker: LLMRanker`, and keyword-only config (`top_k_initial=50`, `top_k_final=10`, `similarity_threshold=0.6`, `use_llm_reranking=True`); `retrieve(query_task, filters=None)` runs Stage 1 — `_task_to_feature_vector` converts the task to a 25-dim float32 array (`log1p(n_samples)` at index 0, raw `n_features` at index 1, raw `n_classes` at index 2, zeros for `None` fields), embeds via `CrossDomainEmbedder.embed`, calls `faiss_index.search(embedding, k=top_k_initial)`, returns `[]` on empty results; Stage 2 — batch-fetches candidates with `asyncio.gather(..., return_exceptions=True)`; failed individual fetches are logged at `WARNING` and skipped without aborting the batch; `None` results are dropped; scores below `similarity_threshold` are discarded; optional field-equality `filters` dict applied; Stage 3 — when `use_llm_reranking=True` and surviving candidates exceed `top_k_final`, delegates to `LLMRanker.rerank`; otherwise slices to `top_k_final` and annotates each result as `"vector similarity"`; `retrieve_with_expanded_queries(query_description, query_task)` calls `QueryExpander.expand`, then for the original description and each expansion creates a task variant via `query_task.model_copy(update={"name": description})` and calls `retrieve(query_variant)` — each variant drives a distinct FAISS embedding so expansions are semantically meaningful; merges via `_deduplicate_and_sort` (keeps highest score per `task_id` UUID, returns sorted descending), and returns the top-`top_k_final` deduplicated results; `QueryExpander`, `LLMRanker`, and `HybridRetriever` exported from `orcanet.retrieval`; 35 unit tests across three files — `test_query_expander.py` (12: `_parse_list_from_response` strips numbered lists, dash bullets, asterisk bullets, blank lines, plain lines, and empty input; `QueryExpander.expand` count, non-empty strings, prompt contains `n_expansions` and `query`, empty LLM response returns `[]`, default `n_expansions=3`), `test_ranker.py` (11: `_parse_ranked_list` parses valid JSON tuples, excludes unknown task IDs, strips backtick fences, returns `[]` on invalid JSON and out-of-range scores, returns all tasks for multi-candidate input;
